# 1.¿Qué es una Base De Datos Relacional?
Las bases de datos relacionales son un tipo de base de datos que organiza los datos en tablas formadas por filas y columnas. Cada fila representa un registro único, mientras que las columnas contienen atributos o características de los datos. 

# 2.¿Qué es una Tabla en el contexto Base De Datos Relacionales?
Es una estructura que organiza los datos de manera logica y coherente.Cada fila representa un registro unico ,mientras que cada columna representa un atributo o campo especifico.

# 3.¿Qué es una relación en el contexto de Base De Datos Relacionales?
Una relación es técnicamente una tabla.
Se define como un conjunto de tuplas (filas) que comparten los mismos atributos (columnas). Su nombre proviene de la teoría matemática de conjuntos, donde una relación representa una correspondencia entre diversos dominios de datos.

# 4.¿Qué representa una columna en el contexto de Base De Datos Relacionales?
una columna representa un atributo o una característica específica de los datos que se están almacenando en una tabla.

# 5.¿Qué representa un renglón en el contexto de Base De Datos Relacionales?
(comúnmente llamado fila o técnicamente tupla) representa un objeto único y completo de la vida real dentro de una tabla.
Si la columna es la "característica", el renglón es la instancia específica que posee esas características.

# 6.¿Qué es un motor de base de datos relacional?
Un motor de base de datos relacional (también conocido como Database Engine o Storage Engine) es el componente de software central que se encarga de realizar las operaciones reales en una base de datos.
Si la base de datos es el "almacén" y el SQL es el "idioma" para pedir cosas, el motor es el empleado que físicamente busca, guarda, organiza y protege los productos.

# 7.¿Cuáles son los motores de bases de datos relacionales mas populares?
Los más destacados son MySQL, PostgreSQL, Oracle Database y Microsoft SQL Server. Se prefieren porque:
Son muy utilizados en aplicaciones web y empresas.
Permiten manejar datos de forma segura, organizada y confiable.

# 8.¿Cuál es el motor de base datos ligero que no necesita un servidor?
El motor ligero que no necesita un servidor es SQLite.
A diferencia de los motores más grandes, SQLite funciona de la siguiente manera:
Sin servidor: No requiere un proceso de servidor separado o un sistema para operar, ya que el motor de la base de datos se integra directamente en la aplicación.

# 9.¿Qué significa SQL en el contexto de base datos relacionales?
Si el motor de la base de datos es el "cerebro" que guarda la información, SQL es el idioma que utilizamos para darle órdenes a ese cerebro y obtener los resultados que necesitamos.

# 10.¿Cuáles son los elementos de un sistema de información basado en base de datos relacionales?
Una estructura (tablas), un software de gestión (SGBDR) y un lenguaje (SQL) para servir a los usuarios.
